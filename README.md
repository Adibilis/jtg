# AJTG - Adibilis Java Type Generator

A Maven plugin that generates **TypeScript types**, **Angular services**, and **Zod validation schemas** from Spring REST controllers via Java reflection.

## Features

- Generates TypeScript interfaces, enums, and union types from Java classes
- Generates Angular `@Injectable` services with typed HTTP methods
- Generates Zod schemas with validation rules derived from Jakarta validation annotations
- Supports generics, optionals, collections, maps, file uploads, and pagination
- Extensible writer system via Java `ServiceLoader`

## Modules

| Module | Description |
|---|---|
| `ajtg-core` | Core parsing and type resolution engine |
| `ajtg-maven-plugin` | Maven plugin entry point |
| `ajtg-angular` | Angular service writer |
| `ajtg-zod` | Zod validation schema writer |

## Requirements

- Java 21+
- Maven 3.9+
- Spring Web on the compile classpath of the target project

## Quick Start

Add the plugin to your Spring Boot project's `pom.xml`:

```xml
<plugin>
    <groupId>ch.adibilis.jtg</groupId>
    <artifactId>ajtg-maven-plugin</artifactId>
    <version>1.0.0-SNAPSHOT</version>
    <executions>
        <execution>
            <phase>compile</phase>
            <goals>
                <goal>generate</goal>
            </goals>
        </execution>
    </executions>
    <configuration>
        <basePackages>
            <basePackage>com.example.api</basePackage>
        </basePackages>
        <outputDirectories>
            <outputDirectory>../frontend/src/generated</outputDirectory>
        </outputDirectories>
    </configuration>
    <dependencies>
        <dependency>
            <groupId>ch.adibilis.jtg</groupId>
            <artifactId>ajtg-angular</artifactId>
            <version>1.0.0-SNAPSHOT</version>
        </dependency>
        <dependency>
            <groupId>ch.adibilis.jtg</groupId>
            <artifactId>ajtg-zod</artifactId>
            <version>1.0.0-SNAPSHOT</version>
        </dependency>
    </dependencies>
</plugin>
```

Then run:

```bash
mvn clean package
```

## Configuration

| Parameter | Required | Default | Description |
|---|---|---|---|
| `basePackages` | Yes | — | Packages to scan for `@RestController` classes |
| `outputDirectories` | Yes | — | Directories to write generated files to |
| `clean` | No | `true` | Delete output directories before regenerating |
| `dateAsString` | No | `false` | Map `java.util.Date` to `string` instead of `Date` |
| `headerSuffix` | No | `""` | Suffix appended to the generated file header comment |
| `environmentImportPath` | No | `../../../environments/environment` | Import path for the Angular environment file |
| `customTypeMappings` | No | `{}` | Custom Java-to-TypeScript type mappings |
| `explicitlyMapClasses` | No | `[]` | Additional classes to include in type generation |
| `subModules` | No | `[]` | Sub-modules to include in classpath resolution |
| `paginationOffset` | No | `0` | Offset for paginated response handling |

### Custom Type Mappings

Override how specific Java types are mapped to TypeScript:

```xml
<customTypeMappings>
    <java.time.LocalDate>string</java.time.LocalDate>
    <java.util.UUID>string</java.util.UUID>
</customTypeMappings>
```

### Sub-Modules

If your controllers reference types from sibling Maven modules, add them so their classes and dependencies are included on the classpath:

```xml
<subModules>
    <subModule>api-core</subModule>
    <subModule>api-shared</subModule>
</subModules>
```

## Generated Output

### TypeScript Types (core)

```typescript
// Generated by Adibilis Java Type Generator

import type User from './User';

export default interface UserResponse {
    id: string;
    name: string;
    user?: User;
}
```

### Angular Services (`ajtg-angular`)

```typescript
// Generated by Adibilis Java Type Generator

import { Injectable, inject } from '@angular/core';
import { HttpClient, HttpParams } from '@angular/common/http';
import { Observable } from 'rxjs';
import { environment } from '../../../environments/environment';
import type UserResponse from '../types/UserResponse';

@Injectable({
    providedIn: 'root',
})
export class UserService {
    baseURL = environment.serverUrl;

    private http = inject(HttpClient);

    getUser(id: string): Observable<UserResponse> {
        return this.http.get<UserResponse>(
            this.baseURL + `/api/users/${id}`
        );
    }
}
```

### Zod Schemas (`ajtg-zod`)

Jakarta validation annotations are translated into Zod validation chains:

```typescript
// Generated by Adibilis Java Type Generator

import { z } from 'zod';

export const UserResponseModel = z.object({
    id: z.string(),
    age: z.number().int().min(0),
    email: z.string().email(),
    tags: z.array(z.string()).optional().nullable(),
});

export type UserResponse = z.infer<typeof UserResponseModel>;
```

Supported annotations: `@Min`, `@Max`, `@Size`, `@NotBlank`, `@Pattern`, `@Email`, `@NotNull`, `@Nullable`.

## Writer Selection

Include the writer modules as plugin dependencies to control what gets generated:

- **Types only** (default) — no extra dependencies needed
- **Types + Angular services** — add `ajtg-angular`
- **Zod schemas + Angular services** — add `ajtg-zod` and `ajtg-angular`. The Zod writer replaces the default type writer since it generates both schemas and inferred types.

## Note on the `-parameters` Compiler Flag

The plugin uses Java reflection to read method parameter names from your Spring controllers. This requires the `-parameters` flag to be passed to `javac`. **Spring Boot projects already have this enabled** via `spring-boot-starter-parent`, so no extra configuration is needed.

If you're using the plugin outside of Spring Boot, add this to your `maven-compiler-plugin` configuration:

```xml
<plugin>
    <groupId>org.apache.maven.plugins</groupId>
    <artifactId>maven-compiler-plugin</artifactId>
    <configuration>
        <parameters>true</parameters>
    </configuration>
</plugin>
```

The plugin will fail fast with a clear error message if parameter names are not available.
